## 一句话定义
在父类定义算法骨架，把可变步骤延迟到子类实现。

## 关键知识点
- “不变流程”放基类；“可变实现”放子类。
- 适合流程稳定、步骤可替换的场景。
- 可与 `final` 配合锁定主流程，避免子类破坏顺序。

## 图解（Mermaid）
```mermaid
graph TD
A[TemplateMethod] --> B[step1 固定]
B --> C[step2 抽象]
C --> D[step3 钩子可选]
```

## 面试答题模板（30~60秒）
1. 先说定义：在父类定义算法骨架，把可变步骤延迟到子类实现。
2. 再说适用场景与优缺点。
3. 最后给一个 C++ 落地点（接口抽象、智能指针、生命周期管理）。

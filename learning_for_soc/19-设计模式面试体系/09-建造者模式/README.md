## 一句话定义
将复杂对象的构建过程拆成多个步骤，按顺序组装。

## 关键知识点
- 适合“参数多、步骤多、顺序敏感”的对象创建。
- Builder 关注“怎么一步步构建”，Factory 更关注“创建哪一类对象”。
- 链式 Builder 需在 `build()` 前做完整性校验。

## 这种模式的好处
- 构建过程更可读，调用方不容易传错参数。
- 同一构建流程可产出不同表示，复用组装逻辑。
- 减少巨型构造函数导致的可维护性问题。

## 实际例子（面试可直接复述）
SQL 查询构建器：按 `select -> from -> where -> orderBy` 逐步拼装，最后 `build()` 生成 SQL 字符串。

## 流程图（Mermaid）
```mermaid
flowchart LR
A[QueryBuilder] --> B[select()]
B --> C[from()]
C --> D[where()]
D --> E[orderBy()]
E --> F[build()]
```

## 面试答题模板（30~60秒）
1. 先下定义：将复杂对象的构建过程拆成多个步骤，按顺序组装。
2. 再讲一个真实业务例子，说明“为什么要用它”。
3. 最后补充优势与边界（什么时候不该用）。

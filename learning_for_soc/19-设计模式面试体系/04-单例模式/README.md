## 一句话定义
确保某个类在系统中只有一个实例，并提供全局访问点。

## 关键知识点
- 常见实现：饿汉式、懒汉式、C++11 局部静态变量。
- 必须禁用拷贝和赋值，防止出现多个实例。
- 面试要点：单例是“权衡方案”，不是默认最佳实践。

## 这种模式的好处
- 统一管理全局共享资源（配置、日志、连接池）。
- 避免重复创建重量级对象，节省资源。
- 访问入口统一，使用成本低。

## 实际例子（面试可直接复述）
配置中心 ConfigManager：程序启动后全局读取同一份配置，避免各模块各自加载造成不一致。

## 面试答题模板（30~60秒）
1. 先下定义：确保某个类在系统中只有一个实例，并提供全局访问点。
2. 再讲一个真实业务例子，说明“为什么要用它”。
3. 最后补充优势与边界（什么时候不该用）。
